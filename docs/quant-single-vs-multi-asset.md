# 单品种与多品种量化交易

量化交易不等于多品种持仓。只交易单一品种也可以是量化交易。

量化的核心是：

```text
用数据 + 规则 + 统计检验 + 自动化执行 来交易
```

所以只交易 BTCUSDT，也可以是量化，例如：

```text
趋势跟随
均值回归
突破策略
波动率择时
网格/马丁改良
盘口做市
资金费率套利
高频 scalping
```

## 单品种量化

优点：

```text
逻辑简单
数据干净
执行容易
更适合先验证策略框架
容易理解盈亏来源
```

缺点：

```text
机会少
策略容易依赖某个市场阶段
回撤可能更集中
容量有限
没有横截面优势
```

## 多品种量化

优点：

```text
机会更多
可以做横截面选币/轮动
风险更分散
因子统计更稳定
可以比较“谁更强”
```

缺点：

```text
系统复杂
数据同步要求高
换手和手续费更重要
容易有幸存者偏差
风控更难
```

## 适用场景

更准确地说：

```text
单品种适合做策略原型和执行型策略
多品种适合做选币、轮动、因子和组合管理
```

如果目标是币安永续选币，多品种更合适，因为优势来自横截面比较：

```text
同一时间，哪些币更强？
哪些币成交额足够？
哪些币波动风险更低？
哪些币资金费率异常？
```

但如果只是想先把交易系统跑通，可以先用单品种，例如 BTCUSDT 或 ETHUSDT，把这些模块跑顺：

```text
数据同步
特征计算
信号生成
回测
手续费/滑点
下单执行
风控
日志
```

然后再扩展到全品种。

## 结论

单品种也可以量化，但多品种更适合做选币和组合收益。这个项目最终应该走多品种，但开发验证可以先从单品种开始。
